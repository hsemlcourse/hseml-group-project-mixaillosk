# Текущий прогресс
## Структура
Объем: 3,475 изображений (train + val) с pixel-level аннотациями
Размер каждого: 2048×1024×3 (RGB)
Общий размер: ~11-12 GB
Классы: 19 оцениваемых (дорога, тротуар, здания, автомобили, пешеходы и т.д.), 35 всего
Классы очень не сбалансированы, о чём говорит их распределение (можно наблюдать в eda.ipynb), так дорожные знаки и люди занимают подавляющую часть всех пикселей на изображениях. Надо будет использовать разные веса для них в функции потерь

Имеются также дополнительные 20000 изображений с масками, однако маски в данных сэмплах сделаны автоматически и имеют погрешность

Также Можно заметить, что официальные SoTA решения на сайте датасета достигаются и без дополнительных данных

## Проблема сплита
Используем официальный сплит CityScapes:
- Train: 2,975 изображений
- Val: 500 изображений
- Test: 1,525 (для финальной оценки на benchmark)

Data leakage предотвращается благодаря:
1. Географическому разделению (разные города в разных сплитах)
2. Отсутствию перекрытия между train/val/test
3. Использованию только fine аннотаций для валидации

## Используемая метрика
Основная метрика: Mean IoU (mIoU):
IoU = TP / (TP + FP + FN)
mIoU = среднее IoU по всем классам

Обоснование выбора:
1. CityScapes имеет сильный class imbalance (дорога занимает ~20% пикселей, а поезда <0.1%)
2. IoU одинаково штрафует ошибки на всех классах, в отличие от accuracy
3. Это официальная метрика benchmark'а, что позволяет сравнивать с state-of-the-art решениями
4. Интуитивно интерпретируема: IoU=0.75 означает 75% пересечение предсказания с ground truth


## Функция потерь (а где CE??)
В качестве функции потерь было решено взять смежную между двумя следующими: Focal Loss и Dice Loss.

Небольшое обоснование данного выбора:
### 1. Cross-Entropy Loss (Кросс-энтропия)

#### Формула
\[
\mathcal{L}_{CE} = -\frac{1}{N} \sum_{i=1}^{N} \sum_{c=1}^{C} y_{i,c} \log(\hat{y}_{i,c})
\]

- \( N \) — общее количество пикселей в батче.
- \( C \) — количество классов (для Cityscapes: 19 классов + фон).
- \( y_{i,c} \) — истинная метка (one-hot encoding, 0 или 1) для пикселя \( i \) и класса \( c \).
- \( \hat{y}_{i,c} \) — предсказанная вероятность (после softmax) для пикселя \( i \) и класса \( c \).

#### Преимущества для Cityscapes
- **Стабильность обучения**: Хорошо работает при относительно сбалансированных данных.
- **Интерпретируемость**: Напрямую максимизирует вероятность правильного класса для каждого пикселя.

#### Недостатки для Cityscapes
- **Проблема дисбаланса классов**: Классы "дорога", "здание" занимают большую часть изображения, в то время как "столб", "светофор", "велосипедист" встречаются редко. Кросс-энтропия “забивает” на редкие классы, стремясь хорошо предсказывать фон и доминирующие объекты.
- **Игнорирование пространственной структуры**: Штрафует каждый пиксель независимо, не учитывая связность предсказанных масок.

### 2. Focal Loss

#### Формула
\[
\mathcal{L}_{Focal} = -\frac{1}{N} \sum_{i=1}^{N} \sum_{c=1}^{C} \alpha_c (1 - \hat{y}_{i,c})^\gamma y_{i,c} \log(\hat{y}_{i,c})
\]

- \( \alpha_c \) — весовой коэффициент для класса \( c \) (обычно обратно пропорционален частоте класса).
- \( \gamma \) — фокусирующий параметр (обычно \( \gamma = 2 \)).
- \( (1 - \hat{y}_{i,c})^\gamma \) — модулирующий фактор: уменьшает вклад "легких" примеров (пикселей, которые модель уже хорошо классифицирует).

#### Преимущества для Cityscapes
- **Борьба с дисбалансом классов на уровне сложности**: Даже если пиксель редкого класса классифицирован правильно, но с низкой уверенностью, он получает большой вес. Это заставляет модель учиться на сложных редких объектах (велосипедисты, знаки), игнорируя простые примеры фона.
- **Автоматическая фокусировка**: Снижает доминирование частых классов без необходимости жёстко настраивать веса.

#### Недостатки для Cityscapes
- **Не оптимизирует метрику IoU напрямую**: Может давать размытые границы между классами, так как всё равно работает попиксельно, а не на уровне маски.

### 3. Dice Loss

#### Формула
\[
\mathcal{L}_{Dice} = 1 - \frac{2 \sum_{i=1}^{N} y_{i,c} \hat{y}_{i,c} + \epsilon}{\sum_{i=1}^{N} y_{i,c} + \sum_{i=1}^{N} \hat{y}_{i,c} + \epsilon}
\]

Обычно усредняется по классам:
\[
\mathcal{L}_{Dice} = 1 - \frac{1}{C} \sum_{c=1}^{C} \frac{2 \sum_{i=1}^{N} y_{i,c} \hat{y}_{i,c} + \epsilon}{\sum_{i=1}^{N} y_{i,c} + \sum_{i=1}^{N} \hat{y}_{i,c} + \epsilon}
\]

- \( \epsilon \) — малое число (например, \( 10^{-7} \)) для предотвращения деления на ноль.
- Остальные обозначения те же.

#### Преимущества для Cityscapes
- **Прямая оптимизация метрики пересечения (IoU)**: Dice Loss максимизирует перекрытие между предсказанием и истиной, что является основной метрикой качества в сегментации.
- **Устойчивость к дисбалансу классов**: Лосс зависит не от количества пикселей класса, а от качества перекрытия. Редкий класс с 10 пикселями и частый с 10 000 пикселей дают сопоставимый вклад.
- **Чёткие границы**: Стремление максимизировать перекрытие часто приводит к более связным маскам, чем у кросс-энтропии.

#### Недостатки для Cityscapes
- **Нестабильность обучения на малых объектах**: При очень маленьком объекте (например, знак на Cityscapes) случайные колебания предсказания могут сильно менять значение Dice Loss, вызывая высокую дисперсию градиента.
- **Плохая калибровка вероятностей**: Модель может научиться выдавать слишком самоуверенные предсказания (близкие к 0 или 1), чтобы максимизировать Dice.

### Итоговое преимущество комбинирования (Compound Loss)

Для Cityscapes наилучший подход — **комбинация Focal Loss и Dice Loss**:
\[
\mathcal{L}_{Total} = \lambda \mathcal{L}_{Focal} + (1 - \lambda) \mathcal{L}_{Dice}
\]

- **Focal Loss** отвечает за стабильное пиксельное обучение, уделяя внимание редким сложным классам и обеспечивая хорошую калибровку (также является надмножеством Cross-Entropy).
- **Dice Loss** оптимизирует финальную метрику IoU и улучшает пространственную связность масок.

В самом коде экспериментов в трейнлупе можно заметить сложение всех трёх лоссов. Изначально предполагалось использование и CE, однако после более глубокого изучения похожих задач выяснилось, что подобная комбинация подходит лишь последовательно, то есть сначала обучение на CE Loss, а после на сумме Focal и Dice Loss

## Модель

В качестве бейзлайна была выбрана модель **MobileNetv2** в качестве бэкбона и добавочная часть для решения задачи сегментации. В качестве более продвинутой модели используется бэкбон **DINOv3** (SoTA прошлого августа среди всех Image-моделей) и сложная архитектура из блоков **ResNet'а** для решения задачи сегментации.
Обе модели были проверены при помощи удобного метода отладки на одном сэмпле для проверки архитектуры и потока градиентов через все слои. Данный метод отчётливо показывает, что не стоит использовать последние слои свёрточных сетей или ViT'ов, поскольку размерность фиче-мап является зачастую слишком маленькой (7 и 32 соответственно для текущих моделей) для решения задачи сегментации, поскольку интерполяция из такой размерности до исходного изображения (128-256-512 и т.д.) очень сильно размывает глобальные признаки (объекты, фон). Этот эффект очень хорошо представлен в части обучения на одном сэмпле бейзлайна: итоговая маска очень сильно размыта в сравнении с реальной.
Полноценное обучение всех моделей будет произведено в CP2
